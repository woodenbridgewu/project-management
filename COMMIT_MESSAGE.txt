## 🎯 主要功能

實作完整的 S3/MinIO 物件儲存服務整合，支援檔案上傳、刪除、預覽功能，
並提供自動回退到本地儲存的機制。

## ✨ 新增功能

### 後端功能
- ✅ 新增 StorageService 統一儲存服務層
  - 支援 AWS S3 和 MinIO 物件儲存
  - 自動檢測配置並初始化 S3 客戶端
  - 支援大檔案上傳（使用 @aws-sdk/lib-storage）
  - 處理非 ASCII 檔名（Base64 編碼）
  - 自動回退到本地儲存機制
  - 檔案刪除功能（同步刪除 S3/MinIO 和資料庫記錄）
  - 預簽名 URL 生成（用於私有 bucket）
  - 檔案代理功能（通過後端訪問）

- ✅ 附件控制器增強
  - 整合 StorageService 進行檔案上傳
  - 支援從 HTTP header 解析檔名（處理中文檔名）
  - 生成預簽名 URL 用於圖片預覽（私有 bucket）
  - 檔案刪除時同步清理 S3/MinIO 儲存
  - 改進錯誤處理和日誌記錄

- ✅ 配置系統擴展
  - 新增 storage 配置區塊
  - 支援環境變數配置（STORAGE_ENABLED, STORAGE_PROVIDER, STORAGE_BUCKET 等）
  - 支援多種儲存提供者（local, s3, minio）

- ✅ 伺服器啟動時儲存服務初始化
  - 顯示儲存服務配置狀態
  - 驗證 S3/MinIO 連接
  - 提供清晰的錯誤提示

### 前端功能

## 🔧 技術細節

### 儲存服務架構
- 統一介面：StorageService 提供統一的 API
- 自動回退：S3/MinIO 失敗時自動使用本地儲存
- 配置驅動：通過環境變數控制行為

### 檔案處理
- 檔名處理：支援中文檔名（Base64 編碼 metadata）

- URL 生成：支援公開 bucket 和預簽名 URL

### 圖片預覽
- 自動識別圖片類型
- 支援預簽名 URL（私有 bucket）
- 支援公開 URL（公開 bucket）
- 錯誤處理和載入優化
